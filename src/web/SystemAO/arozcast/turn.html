How it works
  • Screen share is a direct WebRTC connection between two browsers. Across the Internet the peers are usually behind NAT, so a TURN relay is needed to forward the video.
  • ArozOS runs that relay itself. Credentials handed to the browser are short-lived and signed per session — the relay is not an open proxy.
  • For Internet use, forward the relay port to this host (and set a public IP/hostname with -arozcast_turn_publicip when behind NAT).
  • To use an external TURN service instead, drop an iceServers list at system/arozcast/iceservers.json — it overrides this relay.
Turning the relay off does not break local-network screen share, but remote (Internet) sessions will fail to connect.
